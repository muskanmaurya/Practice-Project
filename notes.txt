################################# user model ######################################

create a model folder in src and then create a new file called user.model.ts in that folder. In that file you can use the following code to create the user model:

always remember first we create the interface of the user which tells us the type of the user fields
and then we create the schema of the user that tells us the structure and validation rules, 
and then we create the model of the user which is an instance of the schema.

check the user model code below:

import mongoose, {Document, Schema} from "mongoose";

export interface IUser extends Document{   //this IUser interface is used to define the structure of the user document in the database. It extends the Document interface from mongoose, which provides additional properties and methods for working with MongoDB documents.
    name: string;
    email:string;
    image:string;
    role:string;
}

const schema: Schema<IUser> = new Schema({  //this schema:Schema<IUser> is used to define the structure of the user document in the database. It specifies the fields and their types, as well as any validation rules or default values.
    name:{
        type:String,
        required:true
    },
    email:{
        type:String,
        required:true
    },
    image:{
        type:String,
        required:true
    },
    role:{
        type:String,
        default:null
    }
},{
    timestamps:true
})

const User = mongoose.model<IUser> ("User", schema); //this is the User model which is of IUser type which is used to interact with the user collection in the database. It provides methods for creating, reading, updating, and deleting user documents.

export default User; //exporting the User model so that it can be used in other parts of the application, such as in controllers or services that handle user-related functionality.


















############################# before creating a user controller we created a middleware for smooth trycatch blocks #######################################################

we don't wanna define these below things again and again in every controller so we can create a middleware for that which will handle the trycatch blocks for us and we can use that middleware in our controllers.
>>>> async(req:Request, res:Response)
>>>> catch(error){
       message: error.message || "An unexpected error occurred"
     }

create a middleware folder in src and then create a new file called trycatch.ts in that folder. In that file you can use the following code to create the asyncHandler middleware:

import {Request, Response, RequestHandler, NextFunction} from "express"; // Importing necessary types from Express

const tryCatch = (handler: RequestHandler): RequestHandler =>{   // Defining a higher-order function named tryCatch that takes a RequestHandler as an argument and returns a new RequestHandler. This function is used to wrap asynchronous route handlers in a try-catch block to handle errors gracefully.
    return async(req: Request, res: Response, next:NextFunction) =>{  // Returning an asynchronous function that takes the request, response, and next function as parameters. This function will execute the provided handler and catch any errors that occur during its execution.
        try{
            await handler(req, res, next); // Awaiting the execution of the provided handler function. If the handler throws an error, it will be caught in the catch block below.
        }catch(error: any){
            res.status(500).json({  
                message: error.message || "An unexpected error occurred"
            })
        }
    }
}  

export default tryCatch;











##################################### another middlware for authnticated users ########################################################


this middlware is only for authtnticated requests which means that the user is logged in and has a valid token. This middleware will check if the token is valid and if it is, it will allow the request to proceed. If the token is not valid, it will return an error response.

export interfacte AuthenticatedRequest extends Request //////when we write this line extends means we are replacing normal request with our authenticated request which we are creating here and this authenticated request will have a user property which will be of type IUser which we created in the user model. So now we can use this authenticated request in our controllers to get the user details from the token.


import {Request, Response, NextFunction} from "express";
import jwt,{JwtPayload} from "jsonwebtoken";
import {IUser} from "../models/User.js";

export interface AuthenticatedRequest extends Request{
    user?: IUser | null;
}

export const isAuth = async(req:AuthenticatedRequest, res: Response, next:NextFunction):  //here first we are creating a middleware function called isAuth which will check if the user is authenticated or not. If the user is authenticated, it will call the next() function to proceed to the next middleware or route handler. If the user is not authenticated, it will return a 401 Unauthorized response.
Promise<void> => {
    try{
        const authHeader = req.headers.authorization;  //we know that the authorization header is used to send the token from the client to the server. So here we are getting the authorization header from the request headers.

        if(!authHeader || !authHeader.startsWith("Bearer ")){   //here we are checking if the authorization header is present and if it starts with "Bearer ". If it is not present or does not start with "Bearer ", we will return a 401 Unauthorized response.
            res.status(401).json({
                message:"Unauthorized: Please Login",
            })
            return;
        }

        const token = authHeader.split(" ")[1];

        if(!token){   //here we are checking if the token is present or not. If it is not present, we will return a 401 Unauthorized response.
            res.status(401).json({
                message:"Unauthorized: token missing",
            })
            return;        
        }

            const decodedValue = jwt.verify(token, process.env.JWT_SECRET_KEY as string) as JwtPayload;   //here we are verifying the token using the jwt.verify() method. If the token is valid, it will return the decoded value. If the token is not valid, it will throw an error which will be caught in the catch block below.

            if(!decodedValue || !decodedValue.user){    //here we are checking if the decoded value is present and if it has a user property. If it is not present or does not have a user property, we will return a 401 Unauthorized response.
                res.status(401).json({
                    message:"Unauthorized: Invalid token",
                })
                return;
            }

            req.user = decodedValue.user;   //here we are setting the user property of the request object to the user property of the decoded value. This will allow us to access the user details in the controllers.
            next();   //here we are calling the next() function to proceed to the next middleware or route handler.
        }catch(error: any){
            console.error("JWT verify error:", error);
            const message = error?.message || "Unauthorized: jwt error";
            res.status(401).json({
                message,
            })
        }
}
